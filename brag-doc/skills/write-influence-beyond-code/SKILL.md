# Write: Influence Beyond Code

This section is optional — omit entirely if no evidence exists.

## What to capture

Leadership that isn't "feature shipped":

- **Mentoring / coaching** — who was mentored, what changed (promotions, ramp time reduced, skill growth)
- **Hiring** — interview loops designed, leveling rubrics, calibration, candidates sourced
- **Standards** — ML evaluation guidelines, coding patterns, platform templates, review processes established
- **Cross-team alignment** — driving consensus across orgs, RFCs, architecture reviews
- **Community** — talks, papers, blog posts, internal workshops, open-source contributions
- **Incident leadership** — major outage response, postmortems, systemic fixes driven

## Exclude

- **Vague mentoring claims** — "mentored junior engineers" without what changed or who benefited
- **Routine job duties** — "participated in code reviews" or "attended architecture meetings" unless you drove a structural change

## Style

Bullet list, ≤6 items. Each bullet: what you did + measurable or observable impact.

## Output

Return only:

```markdown
## Influence Beyond Code

- <bullet>
```
